GDPR / CCPA Data Controls

FasaFas is designed for GDPR and CCPA compliance — with Data Processing Agreements, data subject rights tools, retention policies, privacy controls, and EU data residency options.

GDPR Compliance Controls
  • Data Processing Agreement (DPA): available for all customers — request from legal@fasafas.app
  • Lawful basis: FasaFas processes data under contract (service delivery) and legitimate interest — documented
  • Data minimization: FasaFas collects only data necessary for the service — see Privacy Policy for full inventory
  • Data retention: admin configures retention policies per data type (meetings, recordings, messages, files)
  • Third-party processors: full list of subprocessors available at fasafas.app/legal/subprocessors
  • Data transfers: EU personal data processed in EU-West-1 (Ireland) by default — US processing covered by SCCs
Data Subject Rights (DSAR)
👤 Rights Supported
Right of accessUser: Profile → Privacy → Download My Data. Admin: generate for any user.
Right to erasureUser: Profile → Privacy → Delete My Account. Admin can delete any user's data.
Right to rectificationUser can update all personal data fields in Profile settings at any time.
Right to portabilityData export includes: meetings, transcripts, messages, files — in JSON format.
Right to objectUsers can opt out of analytics processing in Profile → Privacy → Preferences.
Right to restrictAdmin can place a user's data in restriction mode — processing limited to storage only.
CCPA & Data Residency
CCPA (California)
  • California residents have: right to know, right to delete, right to opt-out of sale
  • FasaFas does not sell personal data — no opt-out required for data sale
  • CCPA consumer requests submitted to: privacy@fasafas.app
  • Response time: 45 days (extendable by 45 more with notice)
  • Shine the Light: upon request, FasaFas discloses what personal info is shared with third parties for their direct marketing
EU Data Residency
  • Default: data processed in AWS eu-west-1 (Dublin, Ireland)
  • Enterprise: optional data residency in Germany (eu-central-1, Frankfurt)
  • Meeting recordings, transcripts, and messages stored in your configured region
  • Metadata (user accounts, billing) stored in us-east-1 with SCCs covering transfer
  • Data residency verified in: Admin Panel → Security → Data Residency
