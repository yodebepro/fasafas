Session Revocation (All Devices)

View and instantly revoke all active sessions across every device — browser, desktop app, and mobile — from a single security page, with device info and last-active timestamps.

Automatic Revocation
User Session Management
  • Profile → Security → Active Sessions — shows all active login sessions
  • Per session: device name, OS, browser/app version, IP address (approximate location), first login time, last active time
  • Current session: highlighted with 'This device' badge
  • Revoke one session: click 'Sign out' next to any session — that device is logged out immediately
  • Revoke all others: 'Sign out all other devices' — one click to logout everywhere except your current device
  • Global logout: 'Sign out everywhere' — logout of ALL devices including current (use when account may be compromised)
Admin Session Controls
  • Admin Panel → Users → [username] → Active Sessions — admin can see and revoke any user's sessions
  • Revoke all sessions for a departing employee: off-boarding flow → 'Revoke All Sessions' → instant lockout
  • Bulk revoke: select multiple users → Revoke All Sessions (for incident response)
  • Session duration policy: admin sets maximum session lifetime (e.g., force re-login every 8 hours, 24 hours, or 7 days)
  • Idle timeout: auto-logout after N minutes of inactivity — configurable 15 min to 8 hours
  • Session logs: every login, logout, and revocation logged in the audit trail with timestamp and IP
Automatic Session Revocation Triggers
⚡ Auto-Revocation Events
Password changedAll sessions except current device revoked automatically
MFA method changedAll sessions revoked — requires fresh login with new MFA
Admin locks accountAll sessions revoked immediately — user cannot log in again until unlocked
SAML account disabledSession revoked on next token refresh or SSO attempt
Suspicious activity detectedAdaptive MFA triggered — if failed, session revoked
Device passkey deletedSession on that device invalidated
