Version History & Restore

FasaFas saves your whiteboard automatically every 5 minutes and on every major change — browse the full history, preview any snapshot, and restore to any previous state without losing current work.

Auto-Save & Version History
  • Whiteboard auto-saves continuously — no manual save needed
  • Version snapshot created: every 5 minutes of active use, and on major structural changes (delete many objects, etc.)
  • History stored for 60 days (Basic: 7 days · Pro: 30 days · Business+: 60 days · Enterprise: unlimited)
  • History access: toolbar → ⋯ → Version History — opens history panel on the right
  • Timeline: scrollable list of all snapshots with date, time, and who made changes
  • Preview: click any snapshot → canvas shows that version in read-only mode
Restoring a Version
  1. Open Version History from the toolbar
  2. Browse the timeline — preview any snapshot by clicking it
  3. Found the right version? Click 'Restore this version'
  4. Current canvas is auto-saved as a named version ('Before restore — [timestamp]') so you can undo the restore
  5. Canvas resets to the selected version immediately
  6. All collaborators see the restored state — CRDT handles the sync automatically
Named Versions (Milestones)
  • Save a named version: toolbar → ⋯ → Save Named Version → enter a name
  • Named versions are pinned — never auto-deleted even after 60 days
  • Examples: 'Post Sprint Planning', 'Final Architecture v2', 'Before Client Review'
  • Named versions shown with a star (⭐) in the history panel
  • Share a named version: copy a link to that specific snapshot — recipients see the canvas at that point in time
  • Branch from a version: create a new whiteboard from any named version — useful for 'what if' explorations
