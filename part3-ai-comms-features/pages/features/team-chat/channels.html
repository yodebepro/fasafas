Public & Private Channels

Organize team conversations into persistent, searchable channels — public ones anyone can join, private ones by invitation only — with full message history, file sharing, and AI summaries.

Public vs Private Channels
Public Channels
  • Visible to all members in the workspace
  • Anyone can join without an invitation
  • Searchable in the channel directory
  • Good for: #announcements #general #engineering #marketing
  • Message history accessible to all current and new members
  • Admin can set read-only channels (e.g., #announcements)
  • Unlimited members per channel
Private Channels
  • Visible only to invited members
  • New members must be added by an existing member or admin
  • Not searchable in the public channel directory
  • Good for: leadership discussions, confidential projects, HR matters
  • History only accessible to current members — not visible to new members added later
  • Admin can view all private channel messages (Enterprise with audit rights)
Channel Management
  • Create a channel: + icon in sidebar or /channel in any message box
  • Set channel name, description, and purpose (visible in directory)
  • Pin important messages to top of channel
  • Add channel-specific integrations (e.g., GitHub notifications to #engineering)
  • Archive a channel: history preserved but no new messages accepted
  • Convert a public channel to private (irreversible — admin only)
  • Transfer ownership to another member
  • Mute a channel: stop notifications while keeping membership
