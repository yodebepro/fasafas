Live Transcription Pipeline
  1. SFU taps per-participant audio
    Mediasoup provides individual tracks to ai-companion-service — no mixed stream separation needed
  2. VAD segments audio into 3-second chunks
    Voice Activity Detection splits at silence boundaries, preventing mid-word cuts
  3. Whisper large-v3 inference
    Each chunk transcribed with word-level timestamps, detected language, and confidence scores
  4. NLP post-processing
    Punctuation model adds periods and commas. Custom vocabulary corrects product names and acronyms
  5. Stream to all via Socket.io
    Kafka publishes transcript segments → signal-service pushes ai:transcript-update to all participants in under 2 seconds
Speaker Diarization
Automatic Labels (Pyannote.audio 3.0)
  • Assigns 'Speaker 1', 'Speaker 2'… labels to each segment
  • Consistent across the full meeting — same ID from minute 1 to minute 60
  • Supports up to 20 unique speakers per room
  • ~95% segment accuracy on clean audio, single mic per participant
  • Retroactively corrects earlier segments when more voice data confirms identity
Smart Recognition (Name Mapping)
  • Optional 30-second voice enrollment in Profile settings
  • Voice embedding stored encrypted — 192-dim vector, not a recording
  • Matches live speech → replaces 'Speaker 2' with 'Marcus Smith'
  • Works in meetings you attend, not just ones you host
  • Delete anytime: Profile → Smart Recognition → Delete Voice Data
