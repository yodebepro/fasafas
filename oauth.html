Deepfake Detection (Real-Time)

Real-time AI analysis of every participant's video for signs of synthetic media generation — with a confidence-gated host alert, no biometric data stored, and a full audit log for compliance.

Deepfake Detection Engine
🛡️ Technical Specifications
ModelEfficientNet-B4 fine-tuned on FaceForensics++ and DFDC datasets
Analysis rate2 frames per second per participant — minimal compute overhead
Detection signalsGAN artifacts, temporal inconsistency, landmark drift, lighting mismatch, frequency-domain patterns
Threshold85% confidence for 3 consecutive frames before alert fires — reduces false positives
LatencyDetection result available within 500ms of frame capture
PrivacyFrames analyzed in memory — never stored, no biometric data retained
Security Workflow Integration
  1. Detection fires for a participant
    Confidence score > 85% sustained for 3 frames
  2. Host receives a red banner alert (private)
    '⚠️ Potential synthetic video: [Name]. Review and take action.'
  3. Host chooses an action
    Dismiss · Remove participant · Lock meeting · Liveness challenge (wave hand)
  4. Action logged
    All detections and host responses logged in the meeting audit trail
  5. Post-meeting report
    Security report includes detection events — available via API and Admin Panel
Compliance & Governance
  • Audit log: every detection event recorded with: timestamp, participant ID, confidence score, host action taken
  • Audit log exportable: CSV or JSON via Admin Panel → Security → Deepfake Detection Log
  • API access: GET /api/v1/meetings/:id/security-events — includes deepfake detection data
  • Retention: audit log retained for 1 year by default (configurable to 7 years for compliance)
  • GDPR note: detection analysis is in-memory only — no personal data retained from the video frames themselves
  • Admin toggle: Admin Panel → Security → Deepfake Detection → enable/disable per account
