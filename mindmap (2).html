Real-Time Co-Editing

Multiple people can edit the same document simultaneously — with CRDT-based conflict-free sync, live cursor presence, and offline support that merges changes when you reconnect.

Real-Time Co-Editing
  • Open any Doc → share with teammates → they click the link and start editing immediately
  • Every keystroke synced in under 200ms to all co-editors — no refresh needed
  • No locking: multiple people can edit the same paragraph simultaneously
  • Each editor's changes appear in their assigned color with their name label
  • Autosave: every change persisted — no manual Save button
  • Full change history: every edit recorded (see Version History)
  • Works in: Chrome, Firefox, Edge, Safari, iOS, Android
Live Cursors & Presence
  • Every co-editor's cursor is visible as a colored caret with their name
  • Scroll position: see where others are in the document (their section highlighted in the scrollbar)
  • Presence avatars: top-right corner shows who's currently viewing/editing
  • Click any avatar → document scrolls to where they are
  • Typing indicator: '3 people editing' shown at the top of the doc
  • Away state: cursor fades if a user has been idle for 2+ minutes
How Conflicts Are Resolved
  • FasaFas Docs uses Yjs CRDT — the same library as the Whiteboard
  • Text conflicts: if two people type at the same cursor position, characters are interleaved in order of operation timestamp — no data lost
  • Block conflicts: two people deleting the same block simultaneously → block deleted once (idempotent)
  • Offline edits: changes made offline queued as CRDT operations → merged automatically on reconnect
  • Merge order: chronological by operation timestamp — not by network arrival order
