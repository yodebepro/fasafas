Video Codecs

FasaFas supports five video codecs. The right one is selected automatically based on browser, hardware, and network — but understanding each helps when optimizing quality.

Automatic Codec Selection
  1. Browser capability check
    RTCPeerConnection.getCapabilities('video') returns the list of supported codecs and their capabilities (hardware encode support, SVC layers, etc.).
  2. Negotiate with SFU
    SDP offer/answer negotiation: FasaFas prefers AV1 > VP9+SVC > H.264 (HW) > VP9 > VP8. Both sides agree on the highest-priority mutually supported codec.
  3. Apply codec in RTCRtpSender
    setCodecPreferences() API applied before the offer to express FasaFas preference. Supported on Chrome/Edge/Firefox — polyfilled on Safari.
