AI Post-Processing

Within 15–20 minutes of your meeting ending, FasaFas delivers an AI-generated summary, timestamped smart chapters, and a structured action items list — all in the recording viewer.

AI Meeting Summary
What the Summary Contains
  • Meeting Overview: title, date, attendees, duration
  • Key Discussion Points: 3–5 bullet summary of major topics
  • Decisions Made: numbered list of confirmed decisions
  • Action Items: tasks extracted (also separately in action items section)
  • Next Steps: forward-looking items and upcoming milestones
  • Open Questions: unresolved questions from the meeting
  • Generated by Claude Sonnet using the full speaker-labeled transcript
Editing and Sharing
  • Host can edit the summary before sharing — click "Edit" next to any section
  • Share options: link / Slack / email draft / push to Docs
  • Share without the recording: summary card is shareable independently
  • Webhook: meeting.summary.completed event fires with full summary JSON
  • Custom templates available (AI Companion / Custom AI add-on tiers)
Smart Chapters
  • 5–10 chapters auto-generated per meeting (minimum 3 min per chapter)
  • Each chapter has: timestamp, title (2–6 words), 1-sentence description
  • Chapters shown as markers on the recording timeline and as a clickable list in the sidebar
  • Click any chapter → recording player jumps to that timestamp
  • Deep link to any chapter: share a URL that opens the recording at that exact moment
  • Adjust automatically if the recording is trimmed (timestamps recalculated)
  • Available via API: GET /api/v1/meetings/:id/summary → smart_chapters array
Action Items in Recordings
  • Each action item linked to the transcript timestamp where it was mentioned
  • Click the timestamp → recording player jumps to that exact moment
  • Assignee suggested based on who was speaking or who was named
  • Due date extracted from natural language: "by Friday" → Friday's date
  • Status tracking: mark items complete directly from the recording view
  • Bulk push: "Add All to Tasks" sends all action items to the Tasks module at once
  • Individual push: "→ Tasks" button per item for selective import
  • Integration push: "→ Jira", "→ Asana" buttons if integrations are connected
