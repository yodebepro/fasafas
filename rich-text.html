Version History

Every change to a doc is saved automatically and stored indefinitely — browse the full edit history, compare versions side by side, and restore any previous state with one click.

Auto-Save & History
  • FasaFas saves every edit continuously — no manual save needed, ever
  • Version History: Doc menu (⋯) → Version History → right panel opens
  • History shows: timestamp, editor's name, and a short description of what changed (e.g., 'Added 3 paragraphs · Ana')
  • History stored: 30 days (Basic) · 90 days (Pro) · Unlimited (Business+ and Enterprise)
  • Preview: click any version → doc body shows that version in read-only — without leaving the current version
  • Restore: 'Restore this version' → current doc replaced; a checkpoint is auto-created first so you can undo
Comparing Two Versions
  • Select any two versions in history panel → 'Compare' button
  • Split view: left side shows older version, right side shows newer version
  • Diff highlighted: added text in green, removed text in red, unchanged text in grey
  • Inline diff: toggle to unified (single-pane) view if preferred
  • Copy specific text from the old version back into the current version manually
  • Export diff: download the comparison as a formatted PDF for review
Named Checkpoints
  • Save a checkpoint: Doc menu → Save Version → enter a name
  • Named checkpoints pinned forever — not subject to retention policy
  • Examples: 'Before board review' · 'Sent to legal' · 'v1.0 published'
  • Share a checkpoint link: anyone with doc access can open that specific version
  • Branch from checkpoint: create a new doc starting from that state
  • Compliance: named checkpoints with audit log satisfy document retention requirements for regulated industries
