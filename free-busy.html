Multiple Booking Pages (Business+)

Create separate booking pages for different services, meeting types, or durations — each with its own URL, questions, availability, and confirmation emails. Available on Business+ plans.

Different ServicesPer-DurationPer-Meeting-TypeBusiness+Unique URLs
Multiple Booking Pages
Use Cases
Page Management
Multiple Booking Page Types
  • Booking Pages: Calendar → Booking Pages → New Page
  • Each page has a unique URL: fasafas.app/book/yourname/product-demo
  • Independent settings per page: duration, availability, questions, confirmation email
  • Share a different URL for each meeting type — no confusion for the guest
  • Admin can create team-level booking pages (shared across team members)
  • Toggle any page live/offline at any time
📄 Booking Page Variants
One-on-OneStandard 1:1 meeting — most common type
Round-RobinTeam booking — distributes to available member (Business+)
Group BookingFixed number of seats — e.g., a webinar spot or group training session
CollectiveMeeting that requires multiple hosts to all be available simultaneously (e.g., panel interview)
Use Cases for Multiple Pages
Different Services
  • Product Demo (30 min) — fasafas.app/book/sales/demo
  • Technical Deep Dive (60 min) — fasafas.app/book/sales/technical
  • Partnership Discussion (45 min) — fasafas.app/book/bd/partner
  • Support Call (15 min) — fasafas.app/book/support/help
Different Audiences
  • Enterprise page: requires company name + employee count
  • SMB page: lighter intake form, shorter slot
  • Existing customers: pre-filled form, longer availability window
  • Prospects: adds UTM tracking to measure lead source
Page Management
  • All pages listed in Calendar → Booking Pages with: URL, type, bookings this month, status (live/offline)
  • Clone a page: duplicate an existing page and modify — saves setup time
  • Archive: take a page offline without deleting booking history
  • Analytics per page: views, bookings, conversion rate, no-shows
  • Embed any page on your website: copy embed code → paste into any HTML
  • A/B testing: compare conversion rates between two page versions (Enterprise)
