FIDO2 Passkeys & Biometric

Sign in with a passkey — Face ID, Touch ID, or Windows Hello — instead of a password. Passkeys are phishing-resistant, never travel over the network, and can't be stolen from a database breach.

How Passkeys Work (WebAuthn / FIDO2)
  1. User registers a passkey
    Device generates a public/private key pair. Public key sent to FasaFas. Private key stays on device — never leaves.
  2. Login: FasaFas sends a challenge (random bytes)
  3. Device authenticates the user (Face ID, Touch ID, or PIN)
  4. Device signs the challenge with the private key → signature sent to FasaFas
  5. FasaFas verifies the signature using the stored public key — no password, no secret transmitted
Phishing-Resistant
Because the private key never leaves the device and the challenge is signed for a specific domain (fasafas.app), passkeys cannot be phished. A fake site can't trick the device into signing for a different domain.

Managing Passkeys
  • View your passkeys: Profile → Security → Passkeys
  • Register a new passkey: '+ Add passkey' → device prompts for biometric
  • Passkey nickname: give each passkey a name (e.g., 'MacBook Pro', 'iPhone 15')
  • Delete a passkey: click × next to any passkey — useful when selling a device
  • Multiple passkeys: register passkeys on all your devices — any can be used to log in
  • Recovery: if all passkeys lost, use recovery code (shown at registration) to regain access
