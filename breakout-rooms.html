Bandwidth Estimation

Bandwidth Estimation is how the SFU continuously measures how much data each subscriber can receive, so it can route the appropriate simulcast quality layer in real time.

Congestion Signals
REMB — Receiver Estimated Max Bitrate
  • REMB = Receiver Estimated Maximum Bitrate (RFC 4585 extension)
  • Every 1 second, the subscriber's ICE agent measures how many bytes it received vs how many were expected (based on RTP sequence numbers)
  • Packet loss → lower estimated bandwidth → lower REMB value sent to SFU
  • SFU reads REMB → selects the simulcast layer that fits within the reported bandwidth
  • REMB is less accurate for bursty traffic patterns (WiFi interference, cellular)
  • Replaced by TWCC on modern browsers but still used as a fallback
  • Typical REMB update interval: 1 second
TWCC — Transport Wide Congestion Control
  • TWCC = Transport Wide Congestion Control (draft-ietf-rmcat-gcc)
  • More accurate than REMB — uses inter-packet arrival timing, not just packet loss
  • The SFU stamps each RTP packet with a sequence number (transport sequence number)
  • The subscriber reports back the receive timestamp of each packet in RTCP feedback every 100ms
  • SFU calculates inter-arrival time variance — increasing variance = congestion detected
  • Reacts to congestion 2–3× faster than REMB
  • Enables smoother bitrate probing — temporarily increases bitrate to test available headroom
  • Supported: Chrome, Edge, Firefox. Not supported: Safari (falls back to REMB)
