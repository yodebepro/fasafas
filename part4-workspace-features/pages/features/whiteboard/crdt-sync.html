CRDT Real-Time Sync

Every canvas edit syncs to all collaborators in real time using CRDT (Conflict-free Replicated Data Types) — no locking, no conflicts, no version mismatches, even after going offline and reconnecting.

How CRDT Real-Time Sync Works
  • CRDT = Conflict-free Replicated Data Type — a mathematical data structure that merges concurrent edits without conflicts
  • Implementation: Yjs library with FasaFas's y-websocket sync provider
  • Every object (shape, note, connector) is a Yjs shared type — edits modeled as operations, not full state snapshots
  • Two people moving the same object simultaneously: both moves are applied, then resolved by last-write-wins on position
  • Two people editing the same text simultaneously: character-level merging — no data lost
  • All operations broadcast via WebSocket to all connected peers and also persisted to the server
Offline Mode
  • If you lose your internet connection, the whiteboard continues to function locally
  • Changes made offline are queued as CRDT operations
  • When connection restores: queued operations sync automatically — merged with any changes made by others while you were offline
  • No data loss: offline work is never discarded
  • Status indicator: a yellow banner appears when offline; green when reconnected and synced
  • IndexedDB used for local persistence — canvas state survives browser refresh even while offline
