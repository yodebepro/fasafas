End-to-End Encryption (AES-256-GCM)

Enable End-to-End Encryption mode for meetings where even FasaFas servers cannot decrypt your audio and video — keys are negotiated peer-to-peer, never accessible server-side.

How E2EE Works in FasaFas
  • Standard mode: media encrypted with DTLS-SRTP, but the SFU decrypts to route packets (required for mixing and server-side recording)
  • E2EE mode: each participant's media is encrypted with a key the server never sees — the SFU forwards encrypted packets without decrypting
  • Key exchange: WebRTC's Insertable Streams API (W3C standard) — client-side encryption layer added before packets reach the network
  • Encryption algorithm: AES-256-GCM with a 256-bit key
  • Key negotiation: participants exchange public keys over a secure channel, then derive a shared symmetric key using ECDH (X25519)
  • Key rotation: encryption keys rotated every 60 seconds for forward secrecy
What E2EE Protects
Protected by E2EE
  • Audio streams — FasaFas servers cannot hear your conversation
  • Video streams — FasaFas servers cannot see your video
  • Screen share content — encrypted from sender to receiver
  • Chat messages in the meeting (when E2EE mode is active)
NOT Protected by E2EE
  • Meeting metadata: who attended, when, duration — server can see
  • Participant names and display names — visible to server
  • Recording: cloud recording is disabled in E2EE mode (no server access)
  • AI features: AI transcription, summaries, and captions disabled in E2EE mode
Trade-offs & Limitations
  • Cloud recording: disabled — recordings only possible with local recording (saved to host's device only)
  • AI features: all server-side AI (transcription, captions, summaries) disabled — Nemotron on-device model is the exception
  • Max participants: 500 in standard mode vs 50 in E2EE mode (peer-to-peer key exchange doesn't scale beyond this without additional complexity)
  • Phone dial-in: disabled — PSTN callers cannot join E2EE meetings
  • Breakout rooms: limited — breakout rooms require additional key negotiation (supported in E2EE v2, Enterprise plan)
  • Browser support: Chrome 90+, Edge 90+, Safari 15.1+, Firefox 113+
