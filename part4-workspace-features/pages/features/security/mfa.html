TOTP + SMS MFA

Add a second factor to every login — TOTP authenticator app codes (Google Authenticator, Authy) or SMS OTP — with admin enforcement, per-user grace periods, and adaptive MFA based on risk signals.

📲 MFA Options (Ranked by Security)
Passkey (FIDO2)Most secure. Phishing-resistant. Replaces both password and MFA.
TOTP Authenticator App6-digit time-based code. Compatible with Google Authenticator, Authy, 1Password, Microsoft Authenticator.
SMS OTP6-digit code sent to your mobile number. Convenient but SIM-swap risk.
Email OTP6-digit code sent to your email. Good fallback for users without a phone.
Backup codes10 single-use codes generated at MFA setup. Store securely — used when primary MFA unavailable.
Admin MFA Enforcement
  • Admin Panel → Security → MFA → Enforce MFA for all users
  • Grace period: set 0–14 days before enforcement kicks in — gives users time to set up
  • Exemptions: specific users can be exempted (e.g., API-only service accounts)
  • MFA method requirement: require TOTP minimum (disallow SMS-only) for higher security
  • Report: see which users have MFA enabled vs not — download CSV
  • New employee: when a new user is added, they are prompted to set up MFA before first use if enforcement is on
Adaptive MFA (Risk-Based)
  • Adaptive MFA triggers additional verification based on risk signals — even if user has a session
  • Triggers: new device, new location, VPN exit IP change, unusual login time
  • When triggered: user asked to re-verify with their MFA method — even mid-session
  • Risk score factors: device fingerprint match, IP reputation, geo-velocity (impossible travel), time of day
  • Admin configures sensitivity: Low (only extreme anomalies) / Medium / High (any new factor)
  • Log: all adaptive MFA events logged in Admin Panel → Security → Risk Events
