DLP + Information Barriers

Data Loss Prevention policies scan messages, files, and meeting chat for sensitive content — credit card numbers, SSNs, keywords — and apply automatic actions: warn, block, or redact.

DLP Policies
  • Admin Panel → Compliance → DLP → Create Policy
  • Policy components: Condition (what triggers it) + Action (what happens)
  • Conditions: keyword match, regex pattern, built-in classifiers (credit card, SSN, IBAN, passport number)
  • Built-in classifiers: trained ML models for 30+ sensitive data types — no manual regex needed
  • Scope: apply to channels, DMs, meeting chat, file names, or all
  • Action options: Allow (log only), Warn (user sees warning, can override), Block (message not sent), Redact (replace sensitive text with ****)

Information Barriers
  • Information Barriers prevent specific user groups from communicating with each other — for regulatory compliance (e.g., trading desk vs research team at a financial firm)
  • Create barriers: Admin Panel → Compliance → Information Barriers → Define segments + barrier rules
  • Segments: groups of users defined by department, team, or custom attribute
  • Barrier rules: 'Segment A cannot communicate with Segment B' — bidirectional by default
  • Effect: users in conflicting segments cannot: DM each other, add each other to channels, join each other's meetings
  • Admin override: compliance officer can temporarily lift a barrier for authorized exceptions — logged
DLP Reporting & Audit
  • DLP dashboard: Admin Panel → Compliance → DLP → Events
  • Events list: every policy match with: timestamp, user, channel, matched content (redacted), policy triggered, action taken
  • Filter by: date range, policy name, action type, user or channel
  • Export: CSV or JSON for compliance reporting or legal requests
  • Alerts: configure email or Slack notification when DLP events exceed a threshold (e.g., >10 blocks in 1 hour)
  • SIEM integration: DLP events streamed to your SIEM via webhook or Splunk/Sumo Logic connector
