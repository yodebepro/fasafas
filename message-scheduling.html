Mention Types
@ Mention Reference Guide
@usernameNotifies a specific person. They receive a notification badge and a ping — even if they've muted the channel.
@channelNotifies ALL channel members — including those who have muted the channel. Use sparingly for critical announcements.
@hereNotifies only currently ACTIVE (online) members. Less disruptive than @channel — good for quick questions.
@roleNotify everyone with a specific role: @engineers @design @leadership (admin configures roles)
@[groupname]Notify a user group — e.g., @on-call-team. Admin creates user groups in Admin Panel.
Notification Behavior
  • Direct @username: notification even if channel is muted or user is in DND mode (unless DND set to block all)
  • @channel: notification for all members regardless of mute status — a warning prompt appears before you can send
  • @here: notification only to members whose status is Active/Available — respects DND
  • Mobile push: @username and @channel trigger mobile push by default
  • Desktop: banner notification + badge count in sidebar
  • Admin can restrict who can use @channel and @here (e.g., admins only in #announcements)
Mention Analytics
  • View your recent mentions: sidebar → @mentions — shows all messages where you were mentioned
  • Mention count: analytics dashboard shows how many times you were mentioned this week/month
  • Unread mentions badge: shown on the @ icon in the sidebar
  • Mentions in threads: included in the full mention list even if thread is collapsed
  • Admin can see channel-level mention stats: most mentioned users, most used @channel etc.
